# Lina Brand Voice

This guide defines how Lina should sound in product text and operational messaging.

## Voice Profile

Lina voice is:

- Calm
- Safe
- Adult
- Friendly
- Clear

Lina voice is not:

- Pushy
- Hype-based
- Playful
- Ironic
- Loud

## Writing Rules

- Use short, direct sentences.
- Prefer plain words over expressive marketing language.
- Avoid unnecessary exclamation marks.
- Give guidance without commanding tone.
- Reassure users when something fails.
- Keep emotional temperature low and steady.

## Tone Do/Don't

| Do                                             | Don't                                    |
| ---------------------------------------------- | ---------------------------------------- |
| "Welcome. This is your shared place for care." | "Welcome!! This will change everything!" |
| "Here is how to do it."                        | "You have to do this now."               |
| "We could not connect right now. Try again."   | "Oops! Something broke."                 |
| "Delete photo" + helper text                   | "Delete forever now!"                    |

## Microcopy Patterns

Use these patterns as default templates.

### 1. Errors

Principle:

- State what happened
- Keep blame out
- Offer a clear next step

Pattern:

- "We could not `[action]` right now. Try again."

Examples:

- "We could not save changes right now. Try again."
- "We could not connect right now. Try again."

Avoid:

- "Oops!"
- "Something exploded"
- "Fatal error"

### 2. Loading States

Principle:

- Confirm that the app is working
- Avoid dramatic language

Pattern:

- "`[Action in present participle]`..."

Examples:

- "Loading thread..."
- "Saving changes..."
- "Preparing album..."

Avoid:

- "Hold tight, magic is happening!"
- "Working super hard..."

### 3. Empty States

Principle:

- Keep it practical
- Suggest one clear next action

Pattern:

- "No `[content]` yet. `[Simple next step]`."

Examples:

- "No messages yet. Start the first thread."
- "No photos yet. Add a photo to begin."
- "No contacts yet. Add your first contact."

Avoid:

- "It's lonely here"
- "Let's make this fun"

### 4. Onboarding

Principle:

- Explain value calmly
- Set expectations clearly

Pattern:

- "`[What Lina is]`. `[What users can do next]`."

Examples:

- "Welcome. Here you get one safe line between you."
- "Everything about your child, gathered in one place."
- "Set up your shared line to get started."

Avoid:

- "This is amazing!"
- "Get ready for something magical"

### 5. Destructive Actions

Principle:

- Name the action clearly
- Avoid fear tactics
- Explain impact in helper text

Pattern:

- Button: "`[Clear verb + object]`"
- Helper text: "`[What changes after action]`"

Examples:

- Button: "Delete photo"
- Helper text: "This removes the photo from the shared album."

- Button: "Remove contact"
- Helper text: "This contact will no longer appear in your shared list."

Avoid:

- "Delete forever"
- "This cannot be undone!!!"

## Localization Notes

- Keep sentence structure simple for translation.
- Avoid idioms and slang.
- Keep the same emotional level across languages.
- Preserve clarity over stylistic variation.

## QA Checklist for Copy

- Is the text short and clear?
- Is the tone calm and adult?
- Is there any hype wording?
- Is there a clear next step?
- Does destructive text explain impact plainly?
